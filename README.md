# Noctyrium

Noctyrium is a premium, local-first medical school command center for SGU and
medical students. Alpha 1 is centered on the deployable Vite/React web app in
[`web/`](web/) with optional Vercel serverless backend sync.

The app combines:

- Course, term, module, Lecture, DLA, and PQ tracking
- Pass/mastery tracking with Anki-aware study status
- Productivity logging, daily standups, journal, reports, and tasks
- Step 1 / Step 2 prep surfaces
- Resources, prompt library, hub folders, and JSON backup/import
- Optional name-only cloud progress save, cloud backups, restore, and mock AI endpoints

## Alpha 1 Status

- **Version:** `0.1.0-alpha.1`
- **Frontend:** Vite + React + TypeScript in `web/`
- **Backend:** Vercel serverless functions in `api/`
- **Database:** Neon/Supabase/Postgres via `DATABASE_URL`
- **Persistence:** Local Vault first; cloud sync is optional
- **Package:** static web zip and double-clickable macOS wrapper are generated by `npm --prefix web run package`

Alpha 1 opens with a fresh SGU-oriented scaffold. It keeps Terms 1-5 and the
main BPM/SPPM/PPM course shells, but only includes a few example Lecture/DLA/PQ
rows so new users are not inheriting a personal workload.

## Run Locally

Frontend only:

```sh
cd web
npm install
npm run dev
```

Full Vercel-style app with API routes:

```sh
npm install
cp .env.example .env.local
npm run dev
```

Without `DATABASE_URL`, the app still works locally through the browser Local
Vault and JSON backups. Cloud progress save requires a Postgres connection.

## Backend Setup

Recommended production setup:

1. Create a Neon or Supabase Postgres database.
2. Set `DATABASE_URL` in Vercel.
3. Keep `AI_PROVIDER=mock` for Alpha 1.
4. Run the SQL in [`db/migrations/001_initial.sql`](db/migrations/001_initial.sql), or leave `NOCTYRIUM_AUTO_MIGRATE=true` so the serverless API creates the schema on first use.

Core API routes:

- `POST /api/user/login`
- `GET /api/user/:id`
- `GET /api/data/:userId`
- `POST /api/data/:userId`
- `POST /api/data/:userId/backup`
- `GET /api/data/:userId/backups`
- `POST /api/data/:userId/restore/:backupId`
- `POST /api/progress/save`
- `POST /api/progress/load`
- `POST /api/ai/next-move`
- `POST /api/ai/anki-generator`
- `POST /api/ai/step-planner`
- `POST /api/ai/weak-area-analysis`
- `POST /api/ai/daily-report`

Name login is lightweight identity, not secure authentication. For production
multi-user use, migrate to email magic links, OAuth, or passkeys.

## Deploy To Vercel

Use the repository root as the Vercel project root. The root `vercel.json`
builds `web/`, serves `web/dist`, and keeps `/api/*` on serverless functions.

```sh
npm install
npm run build
```

If Vercel shows `404: NOT_FOUND`, the project is usually pointed at the wrong
root or an older deployment. Redeploy from the repository root after committing
the Vercel config.

## Package

```sh
npm --prefix web run package
```

This creates:

- `web/Noctyrium-web-v0.1.0-alpha.1.zip`
- `web/Noctyrium-mac-v0.1.0-alpha.1.zip` on macOS with `swiftc`

Generated packages, `dist/`, cache files, and personal study folders are ignored
and should not be committed.
